Гидрирование (Hydration):

    ReactDOM.hydrate(element, container[, callback]) — используется для "гидрирования" (оживления) уже существующего HTML, который был сгенерирован на сервере. Это полезно для SSR (Server-Side Rendering), когда HTML-код генерируется на сервере, а затем React "оживляет" его на клиенте, добавляя обработчики событий и другие динамические элементы.


Работа с порталами:

    ReactDOM.createPortal(child, container) — позволяет рендерить дочерние элементы в DOM-узел, который находится вне текущей иерархии компонентов. Это полезно для создания модальных окон, тултипов и других элементов, которые должны быть визуально отделены от основного контента.


Фиберы (Fibers) и планировщик (Scheduler)

    Начиная с React 16, ядро React использует новую архитектуру под названием Fiber. Она позволяет React работать асинхронно и приоритизировать задачи (например, рендеринг или обработку пользовательских событий).

    Планировщик (Scheduler) — это часть ядра, которая управляет приоритетами задач и позволяет React реализовывать такие функции, как конкурентный рендеринг (concurrent rendering).
